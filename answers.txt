Q1

When you type a website address like google.com and press Enter, your browser first checks the DNS to find the IP address of the website. Then it sends a request to that server through the internet. The server processes the request and sends back the website data, which your browser displays as a webpage.


Q2

A client is a device or program that requests information, while a server is a system that provides information or services.

* Client: My phone or web browser (Chrome)
* Server: Google’s computer that sends search results


Q3

DNS (Domain Name System) is like the phonebook of the internet. It changes website names like google.com into IP addresses that computers understand. We need DNS because humans use names, but computers use numbers.


Q4

A firewall is a security system that blocks or allows traffic based on rules to protect a network from attacks. A load balancer distributes incoming traffic across multiple servers to prevent overload. The difference is that a firewall focuses on security, while a load balancer focuses on performance and traffic distribution.

Q5
A web server handles requests like serving web pages (HTML, CSS, images). An application server runs backend logic like processing data and connecting to databases. Backend developers work more with application servers because they handle business logic and data processing.

Q6
The client never communicates directly with the database for security and control reasons. An application server acts as a middle layer that validates requests, protects data, and controls how data is accessed. This improves security, scalability, and organization of the system.

CLIENT --> DNS --> FIREWALL --> LOAD BALANCER --> WEB SERVER --> APPLICATION SERVER --> DATABASE
   |         |          |              |               |                |                 |
   |         |          |              |               |                |                 |
   v         v          v              v               v                v                 v

Client: Sends a request from the browser (e.g. typing a website)
DNS: Converts domain name into an IP address
Firewall: Blocks unsafe or unauthorized traffic
Load Balancer: Distributes requests across multiple servers
Web Server: Receives request and forwards it to the application server
Application Server: Processes logic and communicates with the database
Database: Stores and retrieves required data

DATABASE --> APPLICATION SERVER --> WEB SERVER --> LOAD BALANCER --> FIREWALL --> DNS --> CLIENT

Database: Sends data back after retrieval
Application Server: Processes and formats the response
Web Server: Sends response back to network
Load Balancer: Routes response correctly
Firewall: Ensures response is safe
DNS: Helps maintain routing connection
Client: Displays final result in the browser
